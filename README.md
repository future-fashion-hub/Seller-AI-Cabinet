# Seller-Ai-Cabinet

## Запуск через Docker Compose

1. Убедитесь, что Docker установлен и запущен.
2. В корне проекта выполните команду:
	```bash
	docker compose up --build
	```
3. После сборки сервисы будут доступны по адресам:
	- Фронтенд: `http://localhost:5173`
	- Бэкенд: `http://localhost:8080`

Остановить сервисы можно командой:

```bash
docker compose down
```

## Через npm (локально, без Docker)

1. В папке `server` установите зависимости и запустите сервер:
	```bash
	npm i
	$env:port=8080; npm start
	```
2. В папке `client` установите зависимости и запустите фронтенд:
	```bash
	npm i
	npm run dev
	```
3. После запуска сервисы будут доступны по адресам:
	- Фронтенд: `http://localhost:5173`
	- Бэкенд: `http://localhost:8080`

## Как запустить AI-функционал

Для работы функций «Придумать описание» и «Узнать рыночную цену» используется локальная нейросеть `llama3`, запущенная через `Ollama`. Это позволяет генерировать тексты абсолютно бесплатно и оффлайн. 

### Шаг 1. Установка Ollama
Скачайте и установите легковесный сервер Ollama:
- **Windows**: Запустить в терминале PowerShell команду `irm https://ollama.com/install.ps1 | iex` или скачать инсталлятор с [ollama.com/download](https://ollama.com/download)
- **Linux/macOS**: Запустить команду `curl -fsSL https://ollama.com/install.sh | sh`

### Шаг 2. Загрузка модели llama3
После установки Ollama откройте терминал и выполните команду:
```bash
ollama pull llama3
```
*Загрузка займёт некоторое время (размер модели ~4.7 ГБ).*

### Шаг 3. Запуск сервера Ollama
Убедитесь, что сервер запущен. Обычно он запускается автоматически как фоновый процесс или служба OS.
Если вы остановили его вручную, запустите команду:
```bash
ollama serve
```
REST API будет доступен по локальному адресу `http://localhost:11434`.

## Принятые самостоятельные решения

### 1. Добавление поля `id` в ответ `GET /items` (серверная часть)
В изначальном файле `server.ts` (и в описании `ItemsGetOut`) маршрут `GET /items` возвращал список товаров, исключая поле `id`. Однако без идентификатора фронтенд не может корректно сформировать ссылку на карточку товара (переход по роуту `/ads/:id`). В результате при клике уходил запрос на `/items/undefined`, что вызывало на сервере ошибку 400 (Bad Request). Было принято решение исправить маппинг на бэкенде и отдавать `id` в списке, так как это необходимо для работы интерфейса.

### 2. Исправление работы сервера для метода PUT и CORS-запросов
В изначальном коде сервера в настройках CORS не были указаны разрешенные методы (`Access-Control-Allow-Methods`) и заголовки (`Access-Control-Allow-Headers`), а также не обрабатывались предварительные `OPTIONS` запросы (`preflight`). Из-за этого браузер блокировал отправку изменённых данных формы (PUT `/items/:id`). В `server.ts` была добавлена обработка соответствующих CORS-заголовков и перехват `OPTIONS` запросов. Также исправлено чтение переменной окружения `port` для запуска сервера командой вроде `$env:port=8080; npm start`.

### 3. Добавление сортировки по цене на бэкенде
В изначальной реализации схема валидации и логика `GET /items` не поддерживали сортировку по цене, хотя элементы списков обычно могут быть отсортированы по стоимости. Были расширены типы данных в `types.ts` и `validation.ts` для принятия значения `sortColumn: 'price'`, а в `server.ts` добавлена математическая логика сравнения числовых полей цены `price`.

### 4. Тестирование блока изображений через поле `images`
Для тестирования UI-функционала изображений на фронтенде (главное изображение и миниатюры) к нужному элементу в `server/data/items.json` при необходимости добавляется поле `images` с массивом строковых путей (которые нужно поместить в папку `client/public/images`).
Это сценарий именно для тестовых данных. Отдельные доработки серверной логики для этого не требуются: валидация `PUT /items/:id` поле `images` не принимает.

### 5. Навигация между списком и карточкой через стрелки браузера
Навигация на клиенте в целом выполняется через историю браузера (кнопки «назад/вперёд»).
Отдельно для переходов между страницей с объявлениями (`/ads`) и страницей карточки товара (`/ads/:id`) используется тот же подход: возврат и повторный переход выполняются стрелками браузера. Если карточка не найдена, на странице отображаются ошибка и кнопка «Назад» для возврата к списку объявлений.
